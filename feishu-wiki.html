0. 核心概念

术语含义
appId飞书应用ID（机器人本身）
appSecret该 appId 对应的密钥
accountIdOpenClaw 本地账号名（自定义，如 aiquanzi）
open_id用户在某个 appId 下的用户ID（跨应用不通用）
bindings路由规则 (channel + accountId) -> agentId
一句话：accountId 绑定一套 appId/appSecret，而 open_id 必须属于这套 appId 的生态。

1.1 发不出消息（HTTP 400）

典型日志：
- failed to obtain token
- code: 99991661
- Missing access token
根因 appId 和 appSecret 配对错误，导致拿不到 tenant_access_token
验证 
curl -sS -X POST 'https://open.feishu.cn/open-apis/auth/v3/tenant_access_token/internal' \
  -H 'Content-Type: application/json' \
  --data '{"app_id":"你的appId","app_secret":"你的appSecret"}'
结果判断：
- {"code":0,...} = 凭据有效 ✅
- {"code":10014,"msg":"app secret invalid"} = 凭据错误 ❌

1.2 机器人"收不到消息"

常见根因

  1. 对应 accountId 根本不在 channels.feishu.accounts
  2. bindings 没配/配错，消息路由到错误 agent
  3. 该用户未通过 pairing/allowlist
排查 
openclaw config get channels.feishu.accounts --json
openclaw config get bindings --json
openclaw status --json
日志关键字：
- starting feishu[账号]
- feishu[账号]: WebSocket client started
- feishu[账号]: received message from ...

1.3 open_id cross app

典型报错： code: 99992361, msg: open_id cross app
根因：使用了"另一个飞书应用"下的 open_id，open_id 不能跨 appId 使用。
修复：
1. 用当前机器人重新获取目标用户可用ID
2. 或在该应用下先与用户建立会话后再发
3. 确保发送使用正确 accountId

1.4 未配对（access not configured）

提示样例：
- Pairing code: XXXXXXXX
- openclaw pairing approve feishu XXXXXXXX
修复 
openclaw pairing approve feishu <PairingCode>

1.5 目标错误

典型报错：
- Action send requires a target
- Unknown target "xxx" for Feishu
根因：用了群名称/昵称，不是合法 target ID
修复：
- 用户发消息：用 ou_...
- 群发消息：用 oc_...
- 不要直接用"群名/备注名"做 target

2. 标准排障流程

1查配置是否有该账号
openclaw config get channels.feishu.accounts --json
2查路由是否正确
openclaw config get bindings --json
3查凭据是否有效（token接口）
4查通道是否已启动
openclaw status --json
5查日志关键字
rg -n "failed to obtain token|99991661|open_id cross app|received message|WebSocket" ~/.openclaw/logs/gateway.log | tail -120
6做最小化实发验证
openclaw message send --channel feishu --account <accountId> --target <ID> --message "test" --json

3. 多账号最佳实践

  1. 每个飞书机器人单独一个 accountId，不要混用 secret
  2. bindings 必须显式写 accountId -> agentId
  3. 建议配置默认账号，避免路由歧义
  4. 改账号名时用"新建+迁移+删除旧的"，不要直接硬改
  5. 每次改配置后必做：validate + status + 实发 三连检查

4. 配置示例

{
  "channels": {
    "feishu": {
      "enabled": true,
      "defaultAccount": "aiquanzi",
      "accounts": {
        "aiquanzi": {
          "appId": "cli_xxx1",
          "appSecret": "secret1",
          "enabled": true
        },
        "ceo Goldratt": {
          "enabled": true
        }
      }
    }
  },
  "bindings": [
    { "agentId": "aiquanzi", "match": { "channel": "feishu", "accountId": "aiquanzi" } },
    { "agentId": "gold", "match": { "channel": "feishu", "accountId": "ceo Goldratt" } }
  ]
}

5. 常用命令速查

# 查看飞书账号
openclaw config get channels.feishu.accounts --json

# 查看路由绑定
openclaw config get bindings --json

# 配置校验
openclaw config validate

# 状态
openclaw status --json

# 审批配对
openclaw pairing approve feishu <code>

# 发送测试
openclaw message send --channel feishu --account <accountId> --target <id> --message "test" --json
