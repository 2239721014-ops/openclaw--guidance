第二步：下载 Embedding 模型

💡 nomic-embed-text 是一个开源的文本 embedding 模型，用于将文本转换为向量，是实现长期记忆的关键组件。
📥 在终端运行以下命令下载模型： 
ollama run nomic-embed-text
⏳ 首次下载需要一些时间，模型大小约 274 MB。下载完成后会自动进入交互模式，可以输入文本测试。
✅ 验证模型是否正常运行
在 Ollama 交互界面输入任意文本，确认返回 embedding 向量

第三步：配置 OpenClaw

在 openclaw.json 中添加以下配置：

{
  "tools": {
    "memory": {
      "enabled": true,
      "provider": "ollama",
      "model": "nomic-embed-text",
      "endpoint": "http://localhost:11434"
    }
  }
}

📋 配置参数说明

配置项 推荐值 作用
tools.memory.enabled true 开启长期记忆功能
tools.memory.provider ollama 使用本地 Ollama 作为向量模型
tools.memory.model nomic-embed-text Embedding 模型名称
tools.memory.endpoint http://localhost:11434 Ollama 服务地址
⚠️ 重要：配置完成后需要重启 Gateway 才能生效！ 
openclaw gateway restart

# 任务：为 OpenClaw 配置长期记忆功能

## 目标
为所有 Agent 开启长期记忆能力，基于 Ollama + nomic-embed-text 实现本地向量存储。

## 执行步骤

### 第一步：安装 Ollama（如果未安装）

**Mac:**
- 访问 https://ollama.com/download/mac 下载安装
- 或使用 Homebrew: `brew install ollama`

**Windows:**
- 访问 https://ollama.com/download/windows 下载 OllamaSetup.exe
- 双击运行安装

**Linux:**
```bash
curl -fsSL https://ollama.com/install.sh | sh
```

### 第二步：下载 Embedding 模型
```bash
ollama run nomic-embed-text
```

### 第三步：配置 openclaw.json

在 openclaw.json 中添加以下配置：
```json
{
  "tools": {
    "memory": {
      "enabled": true,
      "provider": "ollama",
      "model": "nomic-embed-text",
      "endpoint": "http://localhost:11434"
    }
  }
}
```

### 第四步：重启 Gateway
```bash
openclaw gateway restart
```

## 验证配置
配置完成后，长期记忆功能即可生效。Agent 会自动：
1. 将重要对话内容存储到本地向量数据库
2. 检索历史相关记忆来增强回答
3. 保持跨会话的上下文连贯性

## 注意事项
- 确保 Ollama 服务正在运行（默认端口 11434）
- 首次使用需下载 nomic-embed-text 模型（约 274MB）
- 所有数据存储在本地，无需联网，安全隐私

常见问题

Q: Ollama 安装后无法启动怎么办？
→ 检查端口 11434 是否被占用：`netstat -an | grep 11434`（Mac/Linux）或 `netstat -ano | findstr :11434`（Windows）
Q: 模型下载速度很慢怎么办？
→ 可以配置 Ollama 使用代理，或者从其他镜像源下载
Q: 如何查看 Ollama 正在运行的模型？
→ 运行 `ollama list` 查看已下载的模型
Q: 可以使用其他 embedding 模型吗？
→ 可以，只要是在 Ollama 支持的模型列表中即可。推荐 nomic-embed-text、mxbai-embed-large 等
Q: 长期记忆的数据存储在哪里？
→ 默认存储在 ~/.openclaw/memory/ 目录下
