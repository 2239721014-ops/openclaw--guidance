二、进阶配置（可选）

🚀 语义检索能力

进阶版让你拥有语义检索能力——搜"我老婆"能找到"妻子"。需要安装 Ollama + Embedding 模型。

1. 安装 Ollama

🍎Mac: brew install ollama 或 下载
🪟Windows: 下载
🐧Linux: curl -fsSL https://ollama.com/install.sh | sh

2. 下载 Embedding 模型

ollama run nomic-embed-text (约 274 MB)

3. 配置 OpenClaw

{
  "tools": {
    "memory": {
      "enabled": true,
      "provider": "ollama",
      "model": "nomic-embed-text"
    }
  }
}
⚠️ 记得执行 openclaw gateway restart

三、存储区别

~/.openclaw/workspace-[agent名]/
├── MEMORY.md          ← 你手动写的"简历"
└── memory/            ← AI 自动记录的对话历史
    ├── 2026-03-09.md
    └── 2026-03-10.md
文件版本谁写的作用
MEMORY.md基础版你手动你的"名片"，每次必读
memory/进阶版AI 自动对话历史，向量检索
💡 两者会冲突吗？ 不会！它们是互补关系：MEMORY.md = "必读简历"，memory/ = "参考资料"，对话时 AI 会同时加载两者。

四、向量模型对比

💡 向量模型做什么？ 把文字转成"向量"，让 AI 能语义理解搜索。比如搜"手机"能找到"iPhone"。
模型参数量文件大小特点推荐场景
nomic-embed-text58M274 MB推荐通用
mxbai-embed-large335M8.4 MB高精度追求准确
bge-m3567M3.6 MB多语言中英混合
all-minilm22M33 MB最轻量资源受限
总结：准确度 mxbai > bge > nomic > all-minilm | 速度 all-minilm > nomic > mxbai > bge
推荐：nomic-embed-text（平衡之选）

五、维护

🔄 更换模型

1ollama run [新模型名] 如 mxbai-embed-large
2修改 openclaw.json 中的 model 字段
3openclaw gateway restart

🗑️ 清理记忆

⚠️ 警告：此操作不可恢复！
rm -rf ~/.openclaw/workspace-[agent名]/memory/
💡 MEMORY.md 不会被删除

📈 memoryFlush 调优

场景推荐值说明
话痨6000-8000减少刷写频率
普通8000-12000平衡
记性好4000-6000更频繁记录

七、常见问题

Q: 群聊能用记忆吗？
→ 可以用 memory/，别用 MEMORY.md！隐私会泄露
Q: MEMORY.md 多大合适？
→ 2000-5000 字，太长 AI 会"记不住重点"
Q: AI 怎么知道要"记住这个"？
→ 告诉它"把这个记下来"就行
Q: 能同时运行多个有记忆的 AI 吗？
→ 可以，每个 agent 有独立 workspace
