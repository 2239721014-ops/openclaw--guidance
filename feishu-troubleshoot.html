📋 问题概述

核心问题：飞书账号凭据错配导致无法获取 access_token

典型报错

  • failed to obtain token - 获取 token 失败
  • 99991661 Missing access token - 缺少访问令牌
  • 99992361 open_id cross app - 跨应用发消息（不是 bug，是规则限制）

🔍 典型症状

症状 原因 解决方案
发不出消息 appSecret 和 appId 不匹配 修正凭据
第二个机器人收不到 账号配置被删除，未启动监听 恢复配置并启动
99992361 错误 跨应用发消息 用同一个应用发

🛠️ 排查步骤（最短路径）

1. 看日志关键字

1 运行命令： 
rg -n "failed to obtain token|99991661|open_id cross app|feishu\[" ~/.openclaw/logs/gateway.log | tail -n 80

2. 验证 app_id + app_secret 是否匹配

2 运行命令： 
curl -sS -X POST 'https://open.feishu.cn/open-apis/auth/v3/tenant_access_token/internal' \
  -H 'Content-Type: application/json; charset=utf-8' \
  --data '{"app_id":"你的app_id","app_secret":"你的app_secret"}'
返回结果判断：
  • code:0 = 凭据有效 ✅
  • code:10014 = app_secret 无效 ❌

3. 确认账号是否在配置里

3 检查 channels.feishu.accounts 下是否有这个账号

没有配置就不会启动

4. 确认账号是否真的启动

4 日志中查找：
  • starting feishu[账号]
  • feishu[账号]: WebSocket client started

✅ 做最小发送验证

openclaw message send --channel feishu --account 账号ID --target 目标ID --message "test" --json

⚙️ 配置建议（避免再次踩坑）

  • 每个飞书机器人单独一个 accounts.<id>，不要复用 secret
  • bindings 明确写 accountId -> agentId
  • 不要只改顶层 channels.feishu.appId/appSecret，多账号时优先看 accounts
  • 每次改完配置，立刻做一次 token 校验 + 最小消息发送
💡 关于 open_id cross app 错误

这不是 bug，是飞书的应用隔离规则：

只能用"同一个飞书应用"下的机器人发给"它能识别的用户ID"。不能跨应用拿 open_id 互发。
